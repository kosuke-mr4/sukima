
# タスク管理アプリ 要件定義書

## 1. プロジェクト概要

* **目的:** ユーザーが自身のタスクを「必須」「任意」などの性質で分類し、かつ優先度を持って管理できるツールを提供する。
* **主要コンセプト:**
* **Local First:** サーバー通信を行わず、プライバシーと速度を重視（LocalStorage完結）。
* **Solid & Clean:** 過度な装飾を排した、情報の視認性が高いモダンフラットデザイン。
* **Rational:** 「やるべきこと」と「優先度」を論理的に整理できるUI。



## 2. 機能要件 (Functional Requirements)

### 2.1 タスク管理機能

* **タスク作成:**
* タイトル（必須）
* 詳細メモ（任意）
* **分類ラベル（Category）:** 以下の2つ（またはユーザー定義）を選択可能。
1. Must Do（やるべきこと / 義務）
2. Nice to Have（やらなくてもいいこと / 推奨）


* **優先度（Priority）:** 3段階設定（High, Medium, Low）。


* **タスク一覧表示:**
* 未完了タスクと完了タスクの表示切り替え。
* 分類ラベルごとのグルーピング表示（「やるべきこと」エリアと「やらなくてもいいこと」エリアを視覚的に分ける）。
* 優先度によるソート機能。


* **タスク操作:**
* 完了/未完了のステータス変更（チェックボックス）。
* タスクの編集。
* タスクの削除。



### 2.2 データ保存 (Persistence)

* **LocalStorage:** ブラウザのローカルストレージを使用。
* **保存タイミング:** タスクの追加・変更・削除時に即時保存。ロード時に復元。

### 2.3 設定・ユーティリティ

* **テーマ切り替え:** ダークモード / ライトモードの手動切り替え（システム設定への追従も検討）。

## 3. UI/UXデザイン要件 (Design Requirements)

「AI Slop」を回避し、人間工学に基づいたモダンなフラットデザインを採用します。

### 3.1 デザイン原則 (Anti-AI Slop Strategy)

* **脱・過剰な角丸 (No excessive border-radius):**
* AIデザインにありがちな「全てが角丸」を避ける。ボタンやカードは角丸を小さく（例: `4px` - `6px`）、あるいは直角にし、シャープでプロフェッショナルな印象を与える。


* **高コントラストとタイポグラフィ:**
* ぼんやりしたグレーのテキストを避け、`#1a1a1a` などの濃い黒を使用。
* フォントはシステムフォント（San Francisco, Inter, Roboto, Noto Sans JP）を使用し、可読性を最優先する。


* **ソリッドカラー:**
* 意味のないグラデーションを禁止。フラットな塗りつぶし色を使用する。
* ドロップシャドウは最小限、または使用せず「ボーダー（線）」で区切りを表現する（ネオブータリズムに近いアプローチ）。



### 3.2 画面構成 (Responsive)

* **レイアウト:**
* **PC:** 2カラム構成（左：Must Do、右：Nice to Have）またはリスト表示。
* **Mobile:** 1カラム構成。タブまたはスクロールでカテゴリを行き来する。


* **配色計画 (Palette Example):**
* **Light Theme:**
* Bg: `#FFFFFF`
* Text: `#111111`
* Accent (Must): `#E63946` (赤系・緊急性)
* Accent (Nice): `#457B9D` (青系・冷静)
* Border: `#E0E0E0`


* **Dark Theme:**
* Bg: `#121212`
* Text: `#EEEEEE`
* Surface: `#1E1E1E`
* Border: `#333333`





## 4. データ構造 (Data Schema)

LocalStorageに保存するJSONデータの構造案です。

```json
[
  {
    "id": "uuid-v4-string",
    "title": "要件定義を完了させる",
    "description": "クライアントへの提出用",
    "category": "must_do",  // "must_do" | "nice_to_have"
    "priority": "high",     // "high" | "medium" | "low"
    "isCompleted": false,
    "createdAt": 1715400000000,
    "updatedAt": 1715400000000
  }
]

```

## 5. 技術スタック案 (Tech Stack)

シンプルかつ軽量に保つため、以下の構成を推奨します。

* **HTML5:** セマンティックなマークアップ。
* **CSS3:**
* Tailwind CSS（設定で角丸などを制限して使用）。
* **JavaScript:** Preact。React等は今回の規模では過剰スペックの可能性あり。
